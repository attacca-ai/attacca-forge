# Contributing to Attacca Forge

Thank you for your interest in contributing to Attacca Forge.

## How to Contribute

### Reporting Issues

If you find a bug, have a suggestion, or want to discuss a feature, please [open an issue](https://github.com/attacca-ai/attacca-forge/issues).

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test locally by installing the plugin and running the skills
5. Submit a pull request with a clear description of what you changed and why

### What We're Looking For

- **New variation types** for the stress testing framework (Category A-E)
- **Example specs** for different domains and trust tiers
- **Methodology docs** that explain concepts clearly
- **Bug fixes** in skill definitions (trigger keywords, output format issues)
- **Installation improvements** for different platforms

### What We're Not Looking For

- Domain-specific scenario libraries (these are vertical-specific and maintained separately)
- Implementation code (Forge is a methodology toolkit, not a runtime)
- Changes that add dependencies (skills should be self-contained SKILL.md files)

## Code of Conduct

Be respectful, constructive, and focused on improving the toolkit. We're building tools that help people build better AI systems — that mission should guide all interactions.

## A Note on the Bigger Picture

Attacca is building commercial tools on top of this methodology. Contributors who demonstrate deep understanding of the framework may be invited to early access programs for upcoming tools in the Attacca ecosystem.
